You may be asking yourself, do we really need another front-end framework? And the answer is no, we don't. The reason Fuselage exists is because I needed another front-end framework —because the ones I tried to use there are too heavy, too opinionated, and overwhelming to customize. This fits my needs, and if no one ever uses it but me, I'd be just fine with that.

Read the Docs Report Bugs

How's it different?

Fuselage has no javascript dependencies, compiles on Libsass, and handles some historically problematic elements well. It has responsive tables baked in, a simple and powerful grid system, and typography based on Modular Scale.

It handles responsive forms well, provides some thoughtful helpers, and looks pretty darn good too. All with the goal to get you up and running quickly. Quit rebuilding things over and over, and build websites faster and more consistently.
